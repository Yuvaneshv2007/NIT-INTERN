1. THE CORE CLINICAL PROBLEM (Why this project matters)
-----------------------------------------------------------------------------
"In standard clinical RAG (Retrieval-Augmented Generation), engineers guess
parameters like search weights and prompt texts using simple human intuition.
In safety-critical medicine, this guesswork leads to silent retrieval failures—
such as mixing up standard female breast cancer with rare male or inflammatory
breast cancers. If the LLM isn't robust, this semantic blur can cause the AI
to confidently hallucinate dangerous, incorrect treatment recommendations."

* Major Vulnerability 1: Semantic Blur
  Vector embeddings (like nomic-embed-text) are great at broad semantic topics,
  but poor at hard clinical boundaries. Chunks on inflammatory breast cancer
  and standard early-stage breast cancer share 95% of the same vocabulary,
  causing vector search to rank them equally.

* Major Vulnerability 2: Human Parameter Bias
  A fixed prompt template or standard hybrid search split (alpha = 0.50) is
  rarely optimal across different clinical topics and textbook structures.

3. THE REINFORCEMENT LEARNING ENGINES (Methodology)
-----------------------------------------------------------------------------
We implemented a dual-agent self-optimizing reinforcement learning loop:

* PART A: The Prompt Bandit (Thompson Sampling)
  We modeled prompt selection as a Multi-Armed Bandit with 8 distinct oncology
  personas ("arms"). Each prompt has a Beta distribution: Beta(alpha_i, beta_i)
  tracking successes and failures. Thompson Sampling balances exploration
  (trying new prompts) with exploitation (playing the winning prompt). The
  bandit crowned "Clinical Precise" as the absolute champion.

* PART B: The Retrieval Optimizer (Bayesian Optimization)
  We optimized the 4D continuous search space (alpha, top_k, rrf_k, reranker_top_n)
  using Bayesian Optimization with a Gaussian Process (GP) surrogate model to
  map exactly how search weights impacted final answer rewards.

-----------------------------------------------------------------------------
4. THE EMPIRICAL REWARD FORMULA (How the AI graded itself)
-----------------------------------------------------------------------------
The system is objectively graded using a multi-factor mathematical formula:

  Reward = 0.35 * Sbert_Sim + 0.25 * Faithfulness + 0.20 * Rouge_L 
         + 0.10 * Speed_Bonus + 0.10 * Context_Relevancy

* Sbert_Sim (35%): Cosine similarity between SBERT embedding of generated answer
  and the ground truth reference answer.
* Faithfulness (25%): Custom sentence-level parsing measuring what percentage
  of generated sentences are strictly supported by the retrieved textbook facts.
* Rouge_L (20%): Lexical F1-measure against ground truth facts.
* Speed Bonus (10%): Linear decay bonus favoring faster generation on CPU.
* Context Relevancy (10%): Similarity of retrieved chunks to original query.
